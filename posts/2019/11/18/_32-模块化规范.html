# 模块化开发

# 概念

将项目的所有功能全部封装成独立的模块，模块之间按照一定的关系组合或者调用，实现整个项目的搭建

# 优点

  • 封装（封装优势）
  • 重用性强
  • 耦合低
  • 方便单独扩展

# 作用

为了统一模块化的标准，减少模块之间的兼容问题，方便数据或方法的管理和使用

# 规范

  • CMD：依赖前置（基本废弃）
  • AMD：按需加载
  • commenJs：后台，nodejs
  • ES6：新版本的js在语法层面上实现了模块化

# 应用场景

  • 加载顺序的问题
  • 模块过多导致产生过多的script\
